# Publication Checklist

Do not publish this proof kit until every item is complete.

- [ ] Confirm the company owns or has assigned rights to the materials.
- [ ] Owner-review `LICENSE.md`, `NOTICE.md`, `COMMERCIAL_USE.md`, and
      `ACCESS_REQUEST.md`.
- [ ] Confirm TWO HANDS NETWORK LTD is the correct public rights holder for this
      proof-kit release.
- [ ] Confirm no private engine source code is included.
- [ ] Confirm no raw private artifacts are included.
- [ ] Confirm no secrets, API keys, signing keys, or `.env` files are included.
- [ ] Confirm no private customer, partner, or personal data is included.
- [ ] Replace draft proof text with sanitized evidence summaries.
- [ ] Regenerate `proof-manifest.json` from the private exporter.
- [ ] Confirm the publication gate passes.
- [ ] Record evidence ZIP hashes and report hashes.
- [ ] Archive the exact public release contents privately.
- [ ] Tag the first public proof release.
- [ ] Confirm the private engine repository remains private.

Solicitor review is recommended before high-value commercial licensing, unusual
partner terms, or direct private-engine access. It is not required before
publishing sanitized public proof materials.
