# Five-Minute Reviewer Walkthrough

This walkthrough is for someone seeing CAB for the first time.

## Minute 1: Identify The Boundary

Read the opening of [README.md](../README.md). Confirm that this repository is a
public proof kit, not the private engine and not an open-source release.

## Minute 2: Understand The Claim

Read [WHY_CAB.md](../WHY_CAB.md). The core claim is that CAB evaluates whether
agents reduce downstream harm while preserving useful capability.

## Minute 3: Inspect The Evidence Map

Open [EVIDENCE_MAP.md](../EVIDENCE_MAP.md). Pick one evidence item and follow
the link to its public proof card.

## Minute 4: Check Hashes And Publication Gate

Open [proof-manifest.json](../proof-manifest.json). Confirm that the manifest
records the public files, directory hash, rights holder, and publication-gate
status.

## Minute 5: Decide The Next Action

Choose one of these outcomes:

- public proof is enough for now;
- request a sanitized reviewer bundle;
- request a technical walkthrough;
- start a commercial licensing discussion;
- decline because the public proof does not answer the needed question.

Use [ACCESS_REQUEST.md](../ACCESS_REQUEST.md) for private review or commercial
follow-up.
