# Proof Evidence Checklist

Each public proof should be understandable without engine access.

Include:

- proof name and version
- claim being tested
- benchmark family or workflow
- sanitized input description
- command summary
- validation status
- score or verdict summary
- source package hash, when safe to publish
- manifest hash, when safe to publish
- reproducibility hash, when safe to publish
- static report path or screenshot
- what the proof does not claim
- how to request private verification

Do not include:

- engine source code
- raw private artifacts
- private prompts or customer data
- signing keys or API keys
- unpublished implementation notes
- non-D-drive local paths unless intentionally sanitized
