# Access Request Checklist

Before requesting private access, decide what you need.

## Evaluation Type

- public proof review only
- private technical review
- private package evaluation
- private source review
- benchmark review
- commercial pilot
- production commercial licence

## Information To Provide

- public contact name
- organisation
- role
- GitHub username
- intended use
- requested access level
- whether an NDA can be signed before access
- whether any third-party AI platform, model provider, search engine, or hosted
  service will access the work

Do not post private contact details, NDA documents, or confidential commercial
details in a public issue. Share those only in the private follow-up.

## Review Outcome

An access request can be:

- accepted
- accepted with NDA
- accepted with limited artifacts only
- deferred for more information
- declined

Private access is not automatic.
