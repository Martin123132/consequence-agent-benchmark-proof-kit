# Access Intake Workflow

Use this workflow when someone asks for private CAB review, pilot access, or
commercial evaluation.

## Public Intake

Ask the requester to open an access request using the public issue template or
send equivalent public-safe information:

- requester name or public contact
- organisation
- GitHub username
- intended use
- requested access level
- commercial or noncommercial status
- whether an NDA can be signed before private material is shared

Do not ask for private email addresses, signed NDA documents, confidential
commercial terms, or personal data in a public issue.

## Owner Review

Before sharing anything private, record the decision privately:

- request date
- requester identity
- organisation and role
- requested access level
- allowed access level
- NDA status
- review window
- material shared
- expiry date
- owner decision and notes

Keep this record outside the public proof-kit repository.

## Access Ladder

Prefer the narrowest useful access level:

- public proof review
- sanitized private review bundle
- private technical walkthrough
- time-limited review mirror
- private engine repository access

Direct private engine access should be reserved for high-trust technical
collaboration after written approval and, where needed, NDA completion.

## After Access

When the review window ends:

- remove repository or bundle access
- ask the recipient to delete private copies if the agreement requires it
- record the closure date
- keep the signed NDA and access notes in company records

Commercial use always needs a separate written commercial licence.
