# Private Access Request

CAB private engine access is not automatic. Access may be granted for serious
technical review, pilot evaluation, partnership review, or commercial licensing
discussion.

This page is not an NDA. It explains the process.

## Before Requesting Access

Review the public proof kit first:

- README
- proof evidence checklist
- commercial-use boundary
- public materials license and commercial licensing notice

Then decide what level of review you need.

## Access Levels

### Level 1: Public Proof Review

Use only the public proof kit. No NDA is normally required.

### Level 2: Sanitized Review Bundle

Receive a private bundle containing sanitized reports, hashes, and validation
outputs. This is the preferred route for most reviewers because it does not
grant repository access.

### Level 3: Private Review Mirror

Receive access to a time-limited review mirror that excludes signing keys,
private artifacts, and unrelated research. NDA required.

### Level 4: Private Engine Repository

Receive direct access to the private engine repository. NDA and owner approval
required. This is reserved for high-trust technical collaborators.

## Required Request Details

In the public request, provide:

- GitHub username
- company or institution
- intended use
- requested access level
- commercial or noncommercial status
- whether you need to run the engine or only inspect evidence
- requested review window

In the private NDA follow-up, provide legal name, role/title, work email, and
any signing details needed by the owner or company. Do not post NDA documents,
private email addresses, or confidential commercial details in a public issue.
Commercial license discussions should be directed to the TWO HANDS NETWORK LTD
COO or owner-approved licensing contact after the access request is triaged.

## Important GitHub Access Note

For personal-account private repositories, GitHub collaborator access can allow
both read and write. Read-only private repo access may require a different
repository ownership model or a review mirror.

For that reason, CAB normally prefers sanitized review bundles or review mirrors
over direct engine repository access.

## NDA Rule

If NDA is required, private access is granted only after the signed NDA is
accepted by the owner or company. Signed NDA documents should be stored in
company records, not committed to GitHub.

An owner/company NDA is acceptable for early controlled review when it clearly
defines the confidential material, review purpose, recipient duties, exclusions,
duration, governing law, and return or deletion duties. Solicitor review is
recommended for high-value commercial terms, unusual partner terms, or direct
private-engine repository access.

## Access Expiry

Private access should be time-limited. The owner may remove access after the
review window, after inactivity, or after the purpose of review is complete.
